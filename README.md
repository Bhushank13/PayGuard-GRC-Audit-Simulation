# Multi-Platform GRC Orchestration & Audit Readiness

**Author:** Bhushan Kotiyan | GRC & Cyber Risk Compliance Analyst
**Project Environment:** PayGuard Inc. (Simulated FinTech Infrastructure)
**Target Frameworks:** ISO/IEC 27001:2022 & AICPA SOC 2 (Revised 2022)

---

## Executive Summary
This repository contains the comprehensive architectural documentation and execution evidence for a multi-platform Governance, Risk, and Compliance (GRC) internal audit project. 

The objective of this project was to transition a simulated high-velocity engineering organization away from static, fragmented spreadsheet tracking by building an automated, continuous compliance telemetry pipeline. By bridging **Jira Software** (for agile execution) and **ServiceNow IRM** (for centralized governance), this project demonstrates the deployment of a Common Control Framework (CCF) that eliminates duplicate audit cycles.

## Systems & Architecture
*   **Action Layer:** Jira Software (Advanced CSV Ingestion, Custom Workflows)
*   **Governance Layer:** ServiceNow IRM/GRC (Entity Scoping, Control Compilation)
*   **Target Asset:** AWS Production Cloud Environment

## Repository Artifacts & Documentation Map

This repository is divided into daily operational phases and executive reporting, reflecting a true end-to-end audit lifecycle.

### 1. Architecture & Execution

*   **`GRC_Day3_Execution_Harmonization_Documentation.pdf`**

### 2. Audit Findings & Governance

    *   *Focus:* The final executive readout. Includes the formal ISO 6.1.2 Quantitative Risk Register detailing the 4 critical cloud infrastructure vulnerabilities discovered (KMS misconfigurations, VPC peering risks, etc.) and their calculated SLA remediation timelines.
